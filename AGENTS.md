# AGENTS.md

本仓库的最高目标：让 Codex / AI 在处理命理问题时，优先调用本地排盘引擎获取可审计的排盘事实，而不是自行心算或凭语言模型生成排盘。

## 强制原则

当用户询问以下内容时：

- 紫微斗数排盘、解盘；
- 八字排盘、用神、格局、大运、流年；
- 合盘、取名、择日；
- 任何需要出生年月日时的命理问题；

你必须先判断是否需要排盘事实。如果需要，必须优先调用本仓库命令或库函数。

## 当前支持能力

当前 Phase 1 仅支持紫微斗数排盘：

```bash
npm run ziwei -- --solar YYYY-MM-DD --time-index N --gender 男|女
```

或在代码中调用：

```ts
import { generateZiweiChart } from "./src/index.js";
```

## 禁止行为

- 禁止自行推算命宫、身宫、主星、四化、大限；
- 禁止在未调用引擎的情况下声称“已排盘”；
- 禁止把解释判断写成排盘事实；
- 禁止把八字和紫微混合在同一个未校验输出里；
- 禁止把个人隐私出生信息写入仓库 fixture，除非用户明确要求并确认。

## 回答命理问题的标准流程

1. 收集必要输入：公历/农历、出生日期、出生时间/时辰、性别、出生地、是否真太阳时。
2. 如果是紫微问题，调用 `npm run ziwei` 或 `generateZiweiChart()`。
3. 输出前先核对：`palaces.length === 12`，命宫、身宫、五行局是否存在。
4. 回答时分三层：
   - 排盘事实；
   - 传统术语含义；
   - 分析判断和不确定性。
5. 如果本引擎尚不支持某项排盘，例如八字 Phase 1 尚未实现，要明确说明“当前引擎尚未支持”，不要用 LLM 硬算。

## 时间与边界规则

Phase 1 默认不做真太阳时换算。若用户出生时间接近时辰边界，必须提醒：

- 当前结果基于输入时辰 / timeIndex；
- 若采用真太阳时或早晚子时规则，结果可能变化；
- 需要补充出生地和规则设置后再做最终判断。

## 工程约束

- 所有紫微排盘调用集中在 `src/ziwei/iztroAdapter.ts`；
- 业务层不要直接 import `iztro`；
- 输出结构统一由 `src/ziwei/normalize.ts` 控制；
- 修改排盘输出结构时同步更新测试和文档。
